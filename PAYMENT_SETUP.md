## 📋 Инструкция по настройке

### Шаг 1: Создайте файл `.env`

Скопируйте `.env.example` в `.env` и заполните токенами:

```bash
cp .env.example .env
```

### Шаг 2: Получите токены

#### BOT_TOKEN
1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте токен в `.env`

#### LZT_TOKEN
1. Зайдите на [lzt.market](https://lzt.market)
2. Перейдите в настройки профиля → API
3. Создайте новый токен
4. Скопируйте в `.env`

#### CRYPTOBOT_TOKEN
1. Откройте [@CryptoBot](https://t.me/CryptoBot)
2. Отправьте `/newtoken`
3. Выберите тип токена (обычный)
4. Дайте название токену
5. Скопируйте в `.env`

### Шаг 3: Настройте вебхуки (опционально, для авто-подтверждения)

1. Откройте панель CryptoBot
2. Перейдите в Settings → Webhooks
3. Укажите URL вашего сервера: `https://your-domain.com/webhook/cryptobot`
4. Выберите события: `invoice_paid`

### Шаг 4: Запустите бота

```bash
python main.py
```

## 🔄 Как работает оплата

1. Пользователь выбирает товар → создаётся заказ в БД (статус: pending)
2. Создаётся инвойс CryptoBot со сроком 30 минут
3. Пользователь оплачивает через CryptoBot
4. **Вариант A:** Пользователь нажимает "✅ Я оплатил" → проверка статуса → покупка на LZT
5. **Вариант B:** Вебхук автоматически обновляет статус заказа в БД
6. После успешной покупки на LZT данные выдаются пользователю
7. Статус заказа обновляется на "paid"

## 🐛 Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| "CRYPTOBOT_TOKEN не задан" | Добавьте токен в `.env` |
| "Ошибка создания счёта" | Проверьте токен CryptoBot и подключение к интернету |
| "Аккаунт уже куплен" | Товар недоступен на LZT, верните деньги пользователю |
| Вебхуки не работают | Проверьте URL в панели CryptoBot, убедитесь что сервер доступен извне |
